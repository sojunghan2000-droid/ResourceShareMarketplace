## 1. 아키텍처 개요

```
┌──────────────────────────────────────────────┐
│            Streamlit App (app.py)            │
│  - st.session_state 기반 라우팅/인증 상태       │
│  - 페이지: 로그인 / 자재목록 / 등록 / 신청함     │
│            / 대여관리 / 대시보드 / 관리자        │
└───────────────┬──────────────────────────────┘
                │ supabase-py (REST/RPC)
┌───────────────▼──────────────────────────────┐
│                 Supabase                     │
│  Auth ── users(JWT)                          │
│  Postgres ── organizations/users/materials/  │
│              loans/loan_events/notifications │
│  Storage ── 자재사진, 수령/반납 증빙          │
│  RLS ── 조직·역할 기반 행 접근제어             │
│  RPC(Postgres fn) ── 상태전이 트랜잭션         │
└──────────────────────────────────────────────┘
```

### 설계 원칙
- **자유 채팅방을 만들지 않는다** — 카톡방의 문제(정보가 잡담에 묻힘)를 재현하지 않기 위해, 통신은 자유 채팅이 아니라 ①알림 ②신청 폼 메모로 흡수한다. 잔여 문답은 P3에서 **대여 건별 댓글 스레드**(`loan_messages`, 아래 §3 선택 스키마)로만 허용. 실시간 그룹 채팅 컴포넌트(Supabase Realtime 기반 메신저)는 도입하지 않는다. *(근거: PRD §2.1)*
- **상태 전이는 DB 트랜잭션(RPC)으로만** — 수량 차감·상태 변경·이력 기록을 한 트랜잭션에서 원자적으로 처리(동시 신청 경합 방지).
- **RLS 우선** — 앱 레벨 권한 체크 + DB RLS 이중 방어. 타 조직 데이터 차단.
- **캐시 전략** — 목록·KPI는 `st.cache_data(ttl=30~60s)`, 변경 시 타깃 무효화(기존 MPM 앱 패턴 준용).
- **모바일 우선 UI** — 단일 컬럼, 큰 버튼, 사진 업로드 간소화.

## 3. 데이터베이스 스키마 (DDL 초안)

```sql
-- 협력사(조직)
create table organizations (
  id            uuid primary key default gen_random_uuid(),
  name          text not null,
  type          text default 'partner',     -- 'partner' | 'owner'
  created_at    timestamptz default now()
);

-- 사용자 (Supabase auth.users 와 1:1 매핑)
create table app_users (
  id            uuid primary key references auth.users(id),
  org_id        uuid references organizations(id),
  name          text not null,
  phone         text,
  role          text not null default 'member',  -- 'member' | 'admin'
  status        text not null default 'pending', -- 'pending' | 'active' | 'disabled'
  created_at    timestamptz default now()
);

-- 자재
create table materials (
  id                uuid primary key default gen_random_uuid(),
  org_id            uuid not null references organizations(id),
  owner_user_id     uuid references app_users(id),
  category          text not null,
  name              text not null,
  spec              text,
  unit              text default 'EA',
  qty_total         int  not null check (qty_total >= 0),
  qty_available     int  not null check (qty_available >= 0),
  location          text,
  photos            jsonb default '[]',
  inspection_status text default 'good',   -- 'good' | 'need_check' | 'no_use'
  inspected_at      date,
  expires_at        date,
  lendable_from     date,
  lendable_to       date,
  status            text default 'active', -- 'active' | 'hidden'
  created_at        timestamptz default now(),
  updated_at        timestamptz default now()
);

-- 대여 건
create table loans (
  id              uuid primary key default gen_random_uuid(),
  material_id     uuid not null references materials(id),
  lender_org_id   uuid not null references organizations(id),
  borrower_org_id uuid not null references organizations(id),
  borrower_user_id uuid not null references app_users(id),
  qty             int  not null check (qty > 0),
  purpose         text,
  pickup_date     date,
  due_date        date not null,
  status          text not null default 'REQUESTED',
  -- REQUESTED|APPROVED|ON_LOAN|RETURN_PENDING|RETURNED|REJECTED|OVERDUE
  reject_reason   text,
  -- 수령(③) 증빙
  pickup_photos   jsonb default '[]',     -- Storage 경로 배열
  pickup_sign_url text,                   -- 신청자 서명 이미지
  -- 반납(⑤) 증빙·정산
  return_qty      int,                    -- 실제 회수 수량(부분반납 대비)
  unreturned_qty  int default 0,          -- 미반납/분실 수량(빌려준수량 - return_qty)
  return_photos   jsonb default '[]',
  return_sign_url text,                   -- 보유자 확정 서명 이미지
  return_condition text,                  -- 'good'|'need_check'|'damaged'
  return_note     text,
  requested_at    timestamptz default now(),
  approved_at     timestamptz,
  loaned_at       timestamptz,
  return_requested_at timestamptz,        -- ④ 신청자 반납요청 시각
  returned_at     timestamptz             -- ⑤ 보유자 최종확정 시각
);

-- 대여 이력(불변)
create table loan_events (
  id            bigserial primary key,
  loan_id       uuid not null references loans(id),
  event_type    text not null,    -- request|approve|reject|pickup|return_request|return_confirm|overdue
  actor_user_id uuid references app_users(id),
  payload       jsonb default '{}',
  created_at    timestamptz default now()
);

-- 알림
create table notifications (
  id          bigserial primary key,
  user_id     uuid not null references app_users(id),
  type        text not null,
  ref_loan_id uuid references loans(id),
  message     text not null,
  read_at     timestamptz,
  created_at  timestamptz default now()
);

-- [P3 선택] 대여 건별 댓글 스레드 — 자유 채팅방이 아니라 '특정 대여 건'에만 종속
-- 도입 시에만 생성. MVP에는 포함하지 않음.
create table loan_messages (
  id          bigserial primary key,
  loan_id     uuid not null references loans(id),  -- 반드시 특정 건에 귀속(전역 채팅 불가)
  author_id   uuid not null references app_users(id),
  body        text not null,
  created_at  timestamptz default now()
);
```

### 상태 전이 RPC (핵심 — 원자적 처리 예시)

```sql
-- 신청: 가용수량 차감 + loan 생성 + 이력 기록을 한 트랜잭션으로
create or replace function request_loan(
  p_material_id uuid, p_qty int, p_due date, p_purpose text, p_pickup date
) returns uuid language plpgsql security definer as $$
declare v_loan uuid; v_avail int; v_owner_org uuid;
begin
  select qty_available, org_id into v_avail, v_owner_org
    from materials where id = p_material_id for update;   -- 행 잠금(경합방지)
  if v_avail < p_qty then raise exception '가용 수량 부족'; end if;
  if (select inspection_status from materials where id=p_material_id) = 'no_use'
     then raise exception '사용금지 자재'; end if;

  insert into loans(material_id, lender_org_id, borrower_org_id, borrower_user_id,
                    qty, due_date, purpose, pickup_date)
  values (p_material_id, v_owner_org,
          (select org_id from app_users where id = auth.uid()),
          auth.uid(), p_qty, p_due, p_purpose, p_pickup)
  returning id into v_loan;

  -- 신청 시점엔 예약 차감(정책에 따라 승인 시 차감으로 변경 가능)
  update materials set qty_available = qty_available - p_qty where id = p_material_id;

  insert into loan_events(loan_id, event_type, actor_user_id)
  values (v_loan, 'request', auth.uid());
  return v_loan;
end $$;
```

> 승인/거절/수령/반납도 동일 패턴의 RPC(`approve_loan`, `reject_loan`, `pickup_loan`, `request_return`, `return_loan`)로 구현. 거절·반납 시 `qty_available` 복원.

```sql
-- 반납 최종 확정: 보유자(Lender)만 호출. 실제 회수 수량만큼 복원, 차액은 미반납 기록.
create or replace function return_loan(
  p_loan_id uuid, p_return_qty int, p_photos jsonb,
  p_sign_url text, p_condition text, p_note text
) returns void language plpgsql security definer as $$
declare v_loan loans%rowtype; v_caller_org uuid;
begin
  select * into v_loan from loans where id = p_loan_id for update;
  select org_id into v_caller_org from app_users where id = auth.uid();

  -- 권한: 보유측(lender_org) 사용자만 확정 가능
  if v_caller_org <> v_loan.lender_org_id then
    raise exception '반납 확정 권한 없음(보유자만 가능)';
  end if;
  if v_loan.status <> 'RETURN_PENDING' then
    raise exception '반납요청(RETURN_PENDING) 상태에서만 확정 가능';
  end if;
  -- 증빙 필수: 사진 1장 이상 + 서명
  if jsonb_array_length(coalesce(p_photos,'[]')) < 1 or p_sign_url is null then
    raise exception '반납 사진·서명 증빙 필수';
  end if;
  if p_return_qty < 0 or p_return_qty > v_loan.qty then
    raise exception '회수 수량 오류';
  end if;

  update loans set
    status = 'RETURNED',
    return_qty = p_return_qty,
    unreturned_qty = v_loan.qty - p_return_qty,
    return_photos = p_photos, return_sign_url = p_sign_url,
    return_condition = p_condition, return_note = p_note,
    returned_at = now()
  where id = p_loan_id;

  -- 실제 회수분만 가용 복원(미반납분은 복원하지 않음)
  update materials set qty_available = qty_available + p_return_qty,
         inspection_status = coalesce(p_condition, inspection_status)
  where id = v_loan.material_id;

  insert into loan_events(loan_id, event_type, actor_user_id, payload)
  values (p_loan_id, 'return_confirm', auth.uid(),
          jsonb_build_object('return_qty', p_return_qty,
                             'unreturned', v_loan.qty - p_return_qty));
end $$;
```

> `request_return`(④)은 신청자가 호출 → `ON_LOAN → RETURN_PENDING` 전이 + 보유자 알림만 수행(수량 미복원).

### RLS 정책(개념)
- `app_users`: 본인 행 + 같은 org admin 조회.
- `materials`: 전체 SELECT 허용(검색 목적, 폐쇄망), INSERT/UPDATE는 `org_id = 내 org`만.
- `loans`: lender_org 또는 borrower_org에 속한 사용자만 SELECT. 전이는 RPC(security definer)로만.
- `admin` 역할은 전체 접근 정책 별도 부여.

## 4. 화면 설계 (Streamlit 페이지)

| 페이지 | 주요 컴포넌트 | 권한 |
|---|---|---|
| **로그인/가입** | 이메일 로그인, 가입신청(승인대기) | 전체 |
| **자재 목록(홈)** | 검색바, 카테고리 필터, 가용여부, 카드형 목록, "신청" 버튼 | member |
| **자재 등록** | 폼(카테고리·품목·규격·수량·사진·위치·점검상태·기간) | member |
| **내 자재 관리** | 내 등록 자재 + 들어온 신청 승인/거절, 반납 확인 | member(보유측) |
| **내 신청함** | 내 신청 상태 추적, 수령확인, 반납요청 | member(신청측) |
| **대시보드** | 개인 KPI(내 가용/대여중/연체) | member |
| **관리자** | 사용자 승인, 전체 KPI, 연체목록·독촉, 절감액 리포트, CSV | admin |

### UX 핵심
- 등록 30초 룰: 카테고리·품목·수량·사진1장만 필수, 나머지 선택.
- 자재 카드에 상태 배지(가용/일부가용/대여중) + 점검상태 색상.
- 신청 폼은 모달/expander 1스텝.
- 연체는 빨강 강조 + 상단 배너.

## 6. 핵심 비즈니스 규칙

| 규칙 | 내용 |
|---|---|
| 수량 차감 시점 | 기본: **신청 시 예약 차감**(중복신청 방지). 정책에 따라 승인 시 차감으로 전환 가능 |
| 부분 대여 | qty_available 범위 내 신청 허용 |
| 안전 차단 | `inspection_status='no_use'` 또는 `expires_at < today` → 신청 RPC에서 예외 |
| 연체 처리 | 일배치(또는 조회 시점 계산)로 `due_date < today AND status='ON_LOAN'` → OVERDUE |
| 이력 불변 | loan_events는 INSERT만, UPDATE/DELETE 금지(RLS) |
| **반납 확정 주체** | **보유자(Lender)만** `RETURN_PENDING → RETURNED` 확정. 신청자는 반납 요청(④)까지만 |
| **반납 수량 복원** | `qty_available` 복원은 **보유자 확정 시점**에만. 실제 회수 수량 입력 → 차액은 `미반납` 기록 |
| **증빙 필수** | 수령(③)·반납확정(⑤)은 사진 ≥1장 + 서명 이미지가 있어야 RPC 성공. 누락 시 예외 |
| 절감액 추정 | Σ(카테고리 기준단가 × qty × 대여일수 가중치). 기준단가 = `category_price` 테이블(P2) |

## 7. 연체/알림 배치
- Streamlit은 상시 서버가 아니므로 **조회 시점 파생 계산**을 기본으로:
  - 목록·대시보드 로드 시 `due_date` 비교로 OVERDUE 표시.
- 능동 알림(D-3/D-1/연체)은 P3에서 **Supabase Edge Function + cron(pg_cron)** 또는 외부 스케줄러로 분리.

## 8. 보안·운영
- `secrets.toml`: Supabase URL, anon key(앱), service_role key는 **RPC/서버 로직 한정**.
- RLS 기본 ON, 정책 누락 테이블 없도록 점검.
- Storage 버킷 비공개 + 서명 URL.
- 감사로그(loan_events) 보존, 백업은 Supabase 자동 백업.
- 한글 PDF: `fonts/` 번들 폰트 사용(다운로드 의존 금지) — 기존 런북 체크리스트 준수.

## 10. 테스트 계획
- 단위: RPC 상태전이(경합·수량부족·안전차단) — SQL 테스트.
- 통합: 신청~반납 happy path + 거절/연체 분기.
