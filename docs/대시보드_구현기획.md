## 2. 영역별 구현 상세

### 2.1 KPI 카드 4개 (난이도: 하 — 데이터 있음)
| 카드 | 값 | 아이콘/색 | 데이터 |
|---|---|---|---|
| 가용 자재 | N **종** | ✓ 초록 | `materials` 중 `qty_available>0` (관리자=전체 / 멤버=내 조직) |
| 대여 진행중 | N **건** | 📦 파랑 | `loans` status=ON_LOAN |
| 승인 대기 | N **건** | ⏱ 주황 | `loans` status in (REQUESTED, RETURN_PENDING) |
| 연체 현황 | N **건** | ⚠ 빨강 | `loans` status=OVERDUE |

- 카드 우상단에 **연한 색 원형 배지 + 아이콘**(React Dashboard에 이미 유사 패턴 존재).
- 범위: 관리자=전체, 멤버=본인 조직. (기존 분기 로직 재사용)
- **추가 백엔드 0.**

### 2.2 카테고리별 자재 현황 — 막대차트 (난이도: 중)
- X축: 카테고리(추락방지·가설구조·보호구·화기밀폐·전기안전), 2계열.
- **가용**(네이비) = 카테고리별 Σ`qty_available` / **대여중**(주황) = 카테고리별 현재 대여 수량(ON_LOAN/OVERDUE/RETURN_PENDING의 Σqty).
- 구현: **Altair** 그룹 막대(색 제어 가능). `st.bar_chart`는 색·그룹 제약 → Altair 권장.
- 데이터: `materials`(가용) + `loans`(대여중) 를 카테고리로 집계.
- **추가 백엔드 0.**

### 2.3 추정 절감액 (분기) — 네이비 카드 (난이도: 상 ⚠ 결정 필요)
사진: "13.5 백만원" + 월별 미니 막대(3·4·5·6월, 6월 강조).

**필요한 것 2가지가 현재 없음:**
1. **기준단가** — 절감액 = Σ(자재 기준단가 × 수량 × 대여기간 가중). 단가 출처 미정(PRD §12 Q2).
   - (A) `category_price` 테이블 신규(카테고리별 표준단가) — **권장**
   - (B) `materials.unit_price` 컬럼 추가(자재별)
   - (C) 데모용 하드코딩 단가
2. **월별 추이 데이터** — 현재는 스냅샷만. 월별 막대는 `loans.loaned_at`(또는 requested_at) 월별 집계로 산출 가능하나 **현재 데이터가 희박**(몇 건). 데모로 보이려면 시드 또는 추정 필요.

> 이 카드만 추가 백엔드/결정이 필요. 단가 정책이 정해지면 산식은 단순.

### 2.4 최근 대여 현황 (난이도: 하)
- 최근 N건(역순): 품목·수량 · (차용 조직 ~반납예정) · 상태 배지(대여중/승인됨/신청대기).
- 상태 배지: 점(dot) 스타일 — `대여중`(파랑)·`승인됨`(초록)·`신청대기`(주황).
- "전체 보기 →" → `내 자재 관리`(또는 관리자=전체 대여 관리)로 이동.
- 데이터: `list_all_loans`(관리자) / `list_incoming_loans`(멤버). **추가 백엔드 0.**

## 3. 기술 선택
| 요소 | Streamlit | React |
|---|---|---|
| KPI 카드 | `st.columns` + HTML 카드(아이콘·배지) | 기존 Kpi 컴포넌트 확장 |
| 카테고리 막대 | **Altair**(`st.altair_chart`) 2계열 색 지정 | recharts BarChart |
| 절감액 미니막대 | HTML/CSS 막대 4개(가벼움) | 동일 |
| 최근 대여 | HTML 행 + 상태 dot | map + badge |

> Altair는 Streamlit 기본 동봉 — 추가 설치 불필요.

## 4. 단계 (Phasing)
| 단계 | 범위 | 백엔드 |
|---|---|---|
| **D1 (즉시 가능)** | KPI 4 + 카테고리 막대 + 최근 대여 | 변경 0 |
| **D2 (결정 후)** | 추정 절감액 카드 | `category_price` 테이블 + 단가값 필요 |

→ **D1만으로 사진의 3/4 영역**(KPI·차트·최근대여)을 즉시 재현. 절감액 카드(D2)는 단가 정책 확정 후.

## 5. 확인 필요 (Open)
1. **절감액 기준단가** 출처 — (A)카테고리 표준단가 테이블 / (B)자재별 단가 / (C)데모 하드코딩? *(D2 진행의 전제)*
2. **월별 추이** — 실데이터 집계(희박) vs 데모 시드값? 분기 범위(최근 4개월 고정?).
3. **"가용/대여중" 단위** — 종(자재 수) vs 수량(EA 합)? *(KPI는 종, 차트는 수량 제안)*
4. **절감액 카드 D2를 지금 포함**할지, 우선 D1만 구현할지.

### 권장 결론
- **D1(KPI·카테고리차트·최근대여) 즉시 구현** — 데이터 있고 백엔드 변경 0, 사진의 대부분 재현.
- **D2(절감액 카드)** 는 §5-1 단가 정책만 정해주시면 바로 추가. 빠른 시연을 원하면 (C)데모 하드코딩으로 카드까지 포함 가능.
