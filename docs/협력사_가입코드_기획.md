# 협력사 가입 코드(비밀번호) 기획

| 항목 | 내용 |
|---|---|
| 작성일 | 2026-06-17 |
| 대상 | Streamlit(`app.py`·`core/`) + React(`web/`) + Supabase(컬럼·RPC) |
| 목적 | 회원가입 시 **선택한 협력사의 4자리 코드**를 입력해야 가입 가능(협력사 자가 검증) |
| 맥락 | 가입 **승인 폐지**(즉시 활성)로 빠진 게이트를 **협력사 코드**가 대체 |

---

## 1. 배경

- 현재: 가입 시 협력사를 **선택만** 하면 즉시 활성(승인 없음) → 누구나 임의 협력사로 가입 가능.
- 개선: 협력사마다 **4자리 가입 코드**를 두고, 가입 시 **선택한 협력사의 코드를 입력·일치**해야 가입.
- 효과: 코드를 아는 사람(= 해당 협력사 담당)만 그 협력사로 가입 → **자가 검증 게이트**.

## 2. 동작 흐름

```
가입 폼
 아이디 / 이름 / 이메일
 소속 협력사 [▼ A협력사]
 협력사 코드 [____]   ← 선택한 협력사의 4자리 코드
 비밀번호 [____]
 [가입]
   ↓ register_user(... , 코드)
   코드 == 해당 협력사 join_code ?  ── 불일치 → "협력사 코드가 올바르지 않습니다"
   일치 → 즉시 active 가입
```

## 3. 데이터 모델

`organizations` 에 **가입 코드 컬럼** 추가:
| 컬럼 | 타입 | 설명 |
|---|---|---|
| `join_code` | text | 숫자 4자리. 협력사별 가입 코드(공유 비밀) |

- 코드 정책: **삼성물산(원청) = `9999`**, **협력사 = 임의 4자리(자동 생성)**.
- 기존 협력사: 백필 — 삼성물산 9999, A/B/C협력사 랜덤 4자리.

## 4. 백엔드 변경

| 구분 | 변경 |
|---|---|
| `organizations.join_code` | 컬럼 추가 + 기존 백필 |
| `register_user` | 인자 `p_join_code` 추가 → `organizations.join_code = p_join_code` 검증. 불일치 시 `BAD_CODE` |
| `list_orgs_public` | **변경 없음** — id·name 만 반환(코드 절대 노출 금지) |
| 신규 협력사 추가(관리자) | 생성 시 코드 부여(자동 랜덤 or 입력) |

- 코드는 **RPC 내부에서만 검증** — 클라이언트로 내려보내지 않음(노출 금지).

## 5. UI 변경

| 화면 | 변경 |
|---|---|
| 가입 폼(Login) | 소속 선택 아래 **"협력사 코드" 입력 필드** 추가 |
| 관리자 → 협력사 관리 | 각 협력사에 **코드 표시·변경** + 신규 추가 시 코드 지정. (관리자가 협력사에 코드 공유) |

## 6. 코드 정책 (확정 요청 §8)

- 삼성물산(원청) = **9999** (고정).
- 협력사 = **임의 4자리** — 신규 생성 시 **자동 랜덤 생성**(관리자 화면에서 확인·재발급 가능) 권장.
- 형식: 숫자 4자리(`^\d{4}$`).

## 7. 보안 주의

1. **코드 노출 금지** — `list_orgs_public` 및 어떤 API도 코드를 반환하지 않음. 검증은 RPC 내부.
2. **4자리는 약함**(1만 조합) — 무차별 대입 가능. 게이트 보조 수단으로만. 필요 시 가입 시도 **rate-limit**(후순위).
3. 코드 변경 시 기존 가입자는 영향 없음(가입 시점만 검증).
4. 관리자(협력사 관리)에서만 코드 조회/변경 — RLS `org_admin_all`.

## 8. 확인 필요 (Open)

1. **"삼성물산"이 어느 조직인가?** — 현재 조직은 `원청-안전관리팀`/`A·B·C협력사`.
   - (a) `원청-안전관리팀` = 삼성물산으로 보고 9999, 또는 (b) 조직명을 `삼성물산`으로 변경?
2. **협력사 코드 부여 방식** — (a) 자동 랜덤 4자리(권장) vs (b) 관리자 수동 입력?
3. **코드 변경 권한/주기** — 관리자가 언제든 변경 가능? 재발급 버튼 둘지.
4. **무차별 대입 방어** — 가입 시도 제한을 지금 넣을지(후순위 가능).
5. **삼성물산(원청)도 코드 게이트** 적용 — 9999로 동일 적용(맞는지 확인).

---

### 권장 결론
- `organizations.join_code` 컬럼 + `register_user` 코드 검증으로 **가입 게이트 복원**.
- 삼성물산 9999 고정, 협력사 자동 랜덤 4자리(관리자 화면에서 확인·재발급).
- 코드는 **절대 클라이언트로 노출하지 않고** RPC 내부 검증.
- 결정 §8 1·2 만 정해지면 마이그레이션 + 가입폼/협력사관리 UI(양 앱) 바로 착수.
