## 1. 배경 / 목표

현재 앱은 **좌측 사이드바만** 있고 상단 헤더가 없다. Figma에는 상단 헤더가 있다. 헤더를 추가해:

1. **현재 위치(페이지 제목)** 를 항상 노출 → 길찾기 명확화
2. **알림(Notification)** 진입점 제공 — 우리는 이미 `notifications` 테이블에 상태전이 알림을 쌓고 있으나 **UI에 노출하는 곳이 없다**. 헤더 종(bell)이 그 자리.
3. **사용자/조직/로그아웃** 을 헤더 우측 표준 위치로 이동(사이드바 하단 로그아웃 대체 가능)
4. **모바일 대응** — 좁은 화면에서 사이드바 토글(햄버거)

> 핵심 가치: 헤더는 "알림을 드디어 화면에 띄우는" 기능적 의미가 가장 크다. 단순 장식이 아님.

## 2. 헤더 구성요소 (좌 → 우)

```
┌───────────────────────────────────────────────────────────────────────┐
│ [☰]  자재 목록                              [🔍]   [🔔 3]   [홍길동 ▾]   │
│  ↑      ↑                                    ↑       ↑          ↑        │
│ 모바일  현재 페이지 제목                    검색   알림벨+뱃지  사용자메뉴 │
│ 토글   (+ 부제/조직)                       (선택)            (조직·로그아웃)│
└───────────────────────────────────────────────────────────────────────┘
```

| 영역 | 구성 | 동작 |
|---|---|---|
| **A. 좌측** | (모바일)햄버거 + **현재 페이지 제목** | 햄버거→사이드바 토글. 제목은 활성 메뉴명 |
| **B. 검색(선택)** | 자재 빠른검색 입력 | 입력 시 자재 목록으로 이동+필터 (P2, 선택) |
| **C. 알림 벨** | 🔔 + 미읽음 카운트 뱃지 | 클릭→드롭다운에 최근 알림 N건, "모두 읽음" |
| **D. 사용자 메뉴** | 아바타(이니셜) + 이름 ▾ | 드롭다운: 조직명·역할 표시, **로그아웃** |

### 알림 벨 상세 (핵심)
- 데이터: `notifications` 테이블 (`user_id`, `type`, `ref_loan_id`, `message`, `read_at`, `created_at`)
- 미읽음 수 = `read_at is null` 카운트 → 빨간 뱃지
- 드롭다운: 최근 10건(시간 역순), 각 항목 `message` + 상대시간
- 항목 클릭 → 해당 대여 건 화면으로 이동(있으면) + `read_at` 갱신
- "모두 읽음" 버튼 → 내 미읽음 전체 `read_at = now()`

## 3. 레이아웃 / 반응형

- **데스크톱**: 사이드바(고정 240px) + 우측 영역 상단에 **sticky 헤더(높이 56~60px)**. 헤더는 콘텐츠 폭과 동일, `border-bottom`, 흰 배경, `backdrop-blur`(선택).
- **모바일(≤768px)**: 사이드바는 기본 숨김 → 헤더 좌측 햄버거로 슬라이드 오픈(drawer). 제목은 축약.
- 색/폰트: 기존 토큰 그대로(흰 배경 `--ss-card`, 보더 `--ss-border`, Pretendard, primary `#2563eb`).

## 4. 데이터 연동

| 필요 | 소스 | 비고 |
|---|---|---|
| 미읽음 수 | `select count(*) from notifications where user_id=auth.uid() and read_at is null` | RLS: 본인 것만(이미 정책 있음) |
| 알림 목록 | `notifications` 최근 10, 역순 | 동일 |
| 읽음 처리 | `update notifications set read_at=now()` | RLS `notif_update` 이미 허용 |
| 사용자/조직 | 기존 `profile`(app_users + organizations) | 추가 쿼리 불필요 |

> **백엔드 변경 없음.** 알림 테이블·RLS는 이미 존재. 헤더는 순수 프론트엔드 작업.

## 5. React 구현 설계 (`web/`)

신규/수정 파일:
```
web/src/components/
  Header.tsx           # 헤더 컨테이너 (제목 + 우측 액션들)
  NotificationBell.tsx # 벨 + 뱃지 + 드롭다운 (notifications 조회/읽음)
  UserMenu.tsx         # 아바타 + 드롭다운(조직·역할·로그아웃)
web/src/lib/api.ts     # listNotifications(), unreadCount(), markAllRead(), markRead(id)
web/src/App.tsx        # 레이아웃 변경: <main> 상단에 <Header title=현재메뉴라벨 />
```

- 라우팅 상태(`nav`)에서 현재 메뉴 라벨을 헤더 `title`로 전달.
- 드롭다운은 경량 자체 구현(이미 radix 미사용 방침) 또는 `@radix-ui/react-dropdown-menu` 1개만 추가.
- 폴링: 미읽음 수는 60초 간격 `setInterval` 재조회(또는 Supabase Realtime 구독은 P3).

### App.tsx 레이아웃 변경 개념
```tsx
<main className="flex-1 flex flex-col">
  <Header title={NAV.find(n=>n.key===nav)?.label} profile={profile} />
  <div className="flex-1 overflow-auto">
    <div className="mx-auto max-w-5xl px-6 py-8">{View()}</div>
  </div>
</main>
```

## 6. Streamlit 적용 방안 (`app.py`)

Streamlit은 고정 top bar가 까다로우므로 **본문 최상단에 헤더 행**을 렌더:
- `render_header(user, title)` 함수: `st.columns([6,1,2])` → 좌측 제목/부제, 가운데 알림 `st.popover("🔔 N")`(미읽음 수 라벨), 우측 사용자 `st.popover`(조직·로그아웃).
- 알림 popover 안에 최근 알림 리스트 + "모두 읽음" 버튼.
- CSS로 하단 보더 + sticky 흉내(선택). 사이드바 로그아웃은 유지하거나 헤더로 이전.

## 7. 작업 범위 / 단계 / 공수(예상)

| 단계 | 내용 | 공수 |
|---|---|---|
| H1 | `api`에 알림 4개 함수 + 헤더 골격(제목+사용자메뉴), 로그아웃 이전 | 0.5d |
| H2 | NotificationBell(뱃지·드롭다운·읽음 처리) — React | 0.5d |
| H3 | 모바일 사이드바 토글(drawer) + 반응형 | 0.5d |
| H4 | Streamlit 헤더 행 + 알림 popover | 0.5d |
| H5 | (선택) 헤더 검색, Realtime 실시간 알림 | P3 |

> MVP 헤더(H1~H2)면 "제목 + 알림 + 사용자메뉴"가 완성되어 Figma 헤더의 핵심을 충족.

## 8. 확인 필요 (Open)

1. **헤더에 검색바**를 둘지(B영역) — 자재 목록 화면에 이미 검색이 있어 중복일 수 있음. 둘 경우 "전역 빠른검색" 성격.
2. **로그아웃 위치** — 사이드바 하단 유지 vs 헤더 사용자메뉴로 이전(권장).
3. **알림 클릭 시 이동** 깊이 — 단순 읽음 처리만 vs 해당 대여 건 상세로 점프.
4. Figma 헤더에 **조직 전환/사이트 선택** 같은 추가 요소가 있는지(단일 현장 전제라 불필요로 가정).
5. 픽셀 정합 — 헤더 **스크린샷 1장** 주시면 높이·간격·아이콘셋을 정확히 맞춤.
