## 1. 수정 ① — 가입 승인 대기 제거 (즉시 활성)

**현행**: 가입 → `status='pending'` → 관리자 승인 → `active`.
**수정**: 가입 → **즉시 `active`** (승인 단계 없음).

| 변경 | 내용 |
|---|---|
| `register_user` RPC | `app_users` insert 시 `status='active'`(was pending) |
| 관리자 센터 | **"가입 승인 대기" 섹션 삭제** |
| 소속 | 가입 시 선택값을 그대로 확정(관리자 확정 단계 없음) |

> ⚠️ 영향: 승인 게이트가 사라져 **소속은 자가 신고로 즉시 확정**. 누구나 가입 폼 접근 시 임의 협력사로 활성 계정 생성 가능. 사내/폐쇄망 전제라면 수용 가능하나, **이메일 도메인 제한(②)이 사실상의 1차 게이트**가 됨.

## 2. 수정 ② — 이메일 수집 + Knox 가입불가

**현행**: 로그인 ID/PW(내부 이메일 `id@safeshare.app`). 실제 이메일 미수집.
**수정**: 가입 시 **실제 이메일 수집**(필수). **Knox(삼성 임직원) 계정 도메인은 가입 차단**.

| 변경 | 내용 |
|---|---|
| 데이터 | `app_users.contact_email text` 컬럼 추가(연락처용). 로그인 식별자는 **여전히 ID** |
| 가입 폼 | 이메일 입력 필드 추가(필수) + 안내문구 |
| 검증 | 이메일 형식 + **Knox 도메인 차단**(클라이언트 + `register_user` RPC 이중) |
| 안내문구 | "Knox(삼성 임직원) 계정은 가입할 수 없습니다. **협력사 이메일**로 가입하세요." |

- **로그인은 ID/PW 유지** — 이메일은 연락처·식별 보조용(알림·향후 복구). *(결정 §6-2)*
- **차단 도메인 규칙** 확정 필요 *(결정 §6-1)* — 예: `@samsung.com` 및 서브도메인.

## 3. 수정 ③ — 관리자 비번변경 제외 + 시스템 관리자만 초기화 (결정 반영)

**결정(§6-3 = d+c)**: 일반 **관리자(admin)** 는 타인 비번 변경 불가. 단 **시스템 관리자(sysadmin)** 는 초기화 가능 + **이메일 복구** 활용.

→ **새 등급 "시스템 관리자(sysadmin)" 신설** (일반 admin 보다 상위).

| 변경 | 내용 |
|---|---|
| 데이터 | `app_users.is_sysadmin boolean default false` 추가. 헬퍼 `is_sysadmin()` |
| 관리자 센터(일반 admin) | "비번 변경"·"초기화" 버튼 **삭제** |
| 시스템 관리자 화면 | 사용자 비번 **초기화(1111)** 유지 — `admin_set_password` 의 권한검사를 `is_admin()` → **`is_sysadmin()`** 로 변경 |
| 본인 비번 변경 | 프로필에서 유지 |
| 초기 sysadmin | 현 admin 계정(`admin`/원청) 을 `is_sysadmin=true` 로 승격 |

### 비번 복구 경로 (2단계)
1. **(c) 이메일 복구** — 수집한 `contact_email` 로 재설정 코드 발송. ⚠️ **별도 메일 인프라 필요**(합성 이메일 로그인이라 Supabase native 복구 불가):
   - `password_resets`(token·user·만료) 테이블 + **Supabase Edge Function**(SMTP/메일서비스로 코드 발송) + "비번 찾기" UI.
   - → **Phase 2**(메일 서비스 결정·설정 후). *(결정 §6-3a)*
2. **(d) 시스템 관리자 초기화** — sysadmin 이 `admin_set_password` 로 즉시 초기화. **Phase 1(즉시 가능)** — 이메일 복구 전까지 1차 복구 수단.

## 4. 수정 ④ — 협력사 목록 관리자 등록 → 가입 시 선택

**현행**: 협력사는 seed 고정. 관리자 UI 없음. 가입 드롭다운은 `list_orgs_public`로 표시(이미 동작).
**수정**: **관리자가 협력사를 등록/수정/삭제** → 가입 드롭다운에 자동 반영.

| 변경 | 내용 |
|---|---|
| 관리자 센터 | **"협력사 관리" 섹션 신설** — 목록 + 추가(이름·유형) + 이름 수정 + 삭제 |
| 백엔드 | `organizations` 직접 CRUD(관리자 RLS `org_admin_all` 이미 허용) |
| 삭제 정책 | 참조(사용자·자재) 있으면 **삭제 차단**(FK) → 안내. *(결정 §6-5)* |
| 가입 폼 | 그대로 `list_orgs_public` 사용 → 관리자가 등록한 목록 자동 노출(추가 작업 0) |

## 7. 구현 단계
| 단계 | 내용 | 비고 |
|---|---|---|
| **Phase 1 (즉시)** | ① 즉시활성 · ② 이메일+Knox차단 · ③ sysadmin 등급+비번 권한 이전 · ④ 협력사 CRUD | 메일 인프라 불요 |
| **Phase 2 (후속)** | (c) 이메일 기반 비번 복구 | **메일 서비스 결정 필요**(Edge Function + SMTP/Resend 등) |

---

### 확정 설계 요약
- **즉시 활성 가입**(승인 폐지) + **협력사 관리자 CRUD**(가입 드롭다운 자동 반영).
- **이메일 수집(연락처)** + **`@samsung.com` 차단** + 안내문구. 로그인은 ID/PW 유지.
- **시스템 관리자(sysadmin) 등급 신설** — 일반 관리자는 비번 변경 불가, sysadmin 만 초기화. 현 admin 을 sysadmin 으로 승격.
- 비번 복구: Phase 1 = sysadmin 초기화, Phase 2 = 이메일 복구(메일 인프라 결정 후).
- **Phase 1 은 결정 완료 → 바로 착수 가능.** Phase 2 는 메일 서비스만 정하면 됨.
